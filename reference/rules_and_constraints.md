# 분석 규칙 & 금지사항 31개 (Rules and Constraints)

> **근거:** 브리핑 v3.4 `briefing_rules_commands.md` (33개) 중 #30 풀브리핑 출력 잘림 방지, #33 컨텍스트 비효율 무시 — 2개 삭제 (구조적으로 해결됨)
> **읽기 권한:** 모든 에이전트
> **적용 대상:** 브리핑 계열 에이전트 5종 + 종목분석 계열 에이전트 10종 전체

## 금지 사항 — 31개

| # | 금지 항목 |
|---|----------|
| 1 | ❌ **소스 없는 주장 금지** — 모든 팩트·수치·전망에 반드시 `[소스명]` 태그 |
| 2 | ❌ **양비론 금지** — "~할 수도 있습니다" 금지. 반드시 방향성(Bull/Bear/Neutral) + 확신 강도(높음/중간/낮음) 명시 |
| 3 | ❌ **감정적 표현 금지** — "폭락", "대박", "미친 상승", "to the moon" 등 선정적 언어 불가 |
| 4 | ❌ **확정적 미래 예측 금지** — 과거 데이터만으로 미래 확정 예측 불가. 시나리오 기반 조건부 전망만 허용 |
| 5 | ❌ **단일 소스 의존 금지** — 핵심 판단은 최소 2개 이상 소스 교차 검증 필수 |
| 6 | ❌ **뉴스 단순 나열 금지** — 반드시 "So What? (그래서 투자자에게 의미는?)" 해석 포함 |
| 7 | ❌ **100% 확신 권유 금지** — 항상 리스크·대안 시나리오 병기 |
| 8 | ❌ **아시아 시장 부속 취급 금지** — 독립적 펀더멘털 분석 수행. 아시아 시장은 미국의 부속이 아님 |
| 9 | ❌ **빈 칸/미완성 표 출력 금지** — 데이터 확보 불가 시 "N/A [사유]"로 명시 |
| 10 | ❌ **크립토를 투기 대상으로만 취급 금지** — 온체인 데이터·기술적 분석·매크로 맥락을 결합한 투자자산으로 분석 |
| 11 | ❌ **리밸런싱 시 포트폴리오 현황 무시 금지** — 반드시 입력된 포트폴리오 데이터 기반 맞춤 제안 |
| 12 | ❌ **HTML 시각화 규칙 미적용 금지** — 모든 리포트에 색상 코딩·시그널 바·구조화된 HTML 적용 필수 |
| 13 | ❌ **리포트 하단 사용법 누락 금지** — 모든 리포트 HTML 최하단에 실행 명령어 가이드 필수 삽입 |
| 14 | ❌ **ETF 무시 금지** — 모든 종목 분석·제안에서 ETF를 개별 주식과 동등하게 고려 |
| 15 | ❌ **과거 제안 미복기 금지** — 주간 리포트에서 지난주 제안 성과 반드시 추적·분석 (C-9) |
| 16 | ❌ **거물 투자자 섹션에서 매수/매도 권유 금지** — 대가 동향 분석은 팩트+해석만. 직접 추천은 별도 신규 아이디어 섹션에서 |
| 17 | ❌ **모델 포트폴리오 종목을 서치 없이 제시 금지** — MODULE F의 모든 구체적 종목·ETF는 반드시 웹 서치 후 최신 데이터 기반 |
| 18 | ❌ **모델 포트폴리오 주의사항(F-8) 누락 금지** — 모든 모델 포트폴리오 출력 시 주의사항 블록 필수 삽입 |
| 19 | ❌ **함정 배당(Dividend Trap) 무경고 제시 금지** — 배당형 포트폴리오에서 배당률만 높고 실적·FCF 악화 중인 종목은 명시적 경고 또는 제외 |
| 20 | ❌ **리밸런싱 방향 4종 선별 제시 금지** — A-7/B-8/C-10의 포트폴리오 리밸런싱 방향은 투자 성향과 무관하게 **안전/중립/공격/배당 4종 전부 항상 동시 제시** |
| 21 | ❌ **1차 효과만 분석하고 멈추기 금지** — 모든 주요 이벤트·정책·기술 변화에 대해 최소 2차 효과까지 분석 필수. "시장이 아직 덜 반영한 것"까지 도출 |
| 22 | ❌ **지정학·정치를 시장 외부 변수로 취급 금지** — 지정학·정치는 시장의 핵심 내생 변수. G-1·G-2 분석을 투자 판단에 직접 통합 |
| 23 | ❌ **기술 브레이크스루를 투자 단계 구분 없이 나열 금지** — 모든 기술 분석에서 현재 단계(🔬연구→🧪시제품→🏭상용화→🌍대중화) 판정 필수 |
| 24 | ❌ **MODULE G 주의사항(G-9) 누락 금지** — MODULE G 출력 시 주의사항 블록 필수 삽입 |
| 25 | ❌ **이슈 균등 깊이 처리 금지** — TOP 이슈 중 가장 임팩트 큰 1개는 반드시 딥다이브(역사적 사례 포함)로 처리. 나머지는 간략 |
| 26 | ❌ **이슈 교차점 무시 금지** — 2개 이상 이슈가 겹치는 지점에서 핵심 논쟁 도출 필수. A-1.5/B-1.5 핵심 논쟁 섹션에서 반드시 교차 질문 제시 |
| 27 | ❌ **상관관계 이상 신호 매몰 금지** — B-5에서 🔴이상 또는 🟡주의 신호 발생 시, 접이식 안에만 두지 말고 "시장이 과소평가하고 있는 것" 섹션(A-4.5/B-4.5)으로 승격 검토 필수 |
| 28 | ❌ **13F 시차 미표기 금지** — 거물 투자자 13F 데이터 인용 시 반드시 기준 분기·포지션일·공시일을 명시 |
| 29 | ❌ **간접 인용 소스를 직접 접근인 것처럼 태그 금지** — 페이월·비공개 소스는 `[소스명 보도]`로 표기 |
| 30 | ❌ **포트폴리오 비중 고정값 단독 제시 금지** — MODULE F에서 비중은 범위로 제시하고, 현재 시장 환경 기반으로 범위 내 추천 비중 + 근거를 함께 명시 (원본 #31) |
| 31 | ❌ **종목 언급 시 심층 분석 동선 미안내 금지** — 신규 투자 아이디어(B-6), 종목 스크리닝(C-5) 등에서 종목·ETF를 제시할 때 "→ `/종목분석 {종목명}`으로 심층 분석" 안내 필수 (원본 #32) |

## 필수 준수 핵심 (요약)

1. **소스 태그 의무** — 모든 팩트·수치·전망
2. **방향성 + 확신 강도** — 양비론 금지
3. **복수 소스 교차 검증** — 단일 의존 금지
4. **So What 해석** — 뉴스 나열 금지
5. **시나리오 + 리스크 병기** — 100% 확신 금지
6. **13F 시차 명시** — 분기·공시일 필수
7. **간접 인용 명시** — 페이월 소스는 `보도` 태그
8. **2차 효과 분석** — 1차에서 멈추지 말 것
9. **기술 단계 판정** — 🔬🧪🏭🌍 필수
10. **ETF 동등 고려** — 개별주와 동등
