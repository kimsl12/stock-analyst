$ARGUMENTS 인자로 **리밸런싱**을 실행해줘.

## 명령 정보

- **모듈:** D (briefing_module_CDE.md MODULE D)
- **산출물:** `reports/briefing/rebalancing_{유형}_{YYYYMMDD}.html`
- **목적:** 자산군 수준의 비중 변화 방향 (구체 종목 지정 없음)

## 인자 해석

- 인자 없음 → `all` (4종 모두)
- `안전형` / `중립형` / `공격형` / `배당형` → 해당 1종만
- `--html` → HTML 동시 생성 (기본)

## briefing-lead 에 전달할 컨텍스트

```
mode: rebalancing
target_type: all | 안전형 | 중립형 | 공격형 | 배당형
sections (briefing_module_CDE.md MODULE D):
  - D-1 현재 시장 환경 진단 (매크로 변수 + 방향성)
  - D-2 직전 리밸런싱 대비 변화 자산군 식별
  - D-3 자산군별 비중 변화 표 (직전 → 새 비중 + 매크로 근거)
  - D-4 리스크 메모 (매크로 충격 시 대응)
required_extras:
  - 4종 포트폴리오 방향 (target_type=all 시)
  - debate-card 1건 (자산군 비중 논쟁)
  - contrarian-card 1건 (시장 합의와 다른 자산 배치)
required_writes:
  - knowledge-base/portfolio/model_portfolios.md (CURRENT 갱신)
  - knowledge-base/portfolio/rebalancing_history.md (append, 덮어쓰기 금지)
  - knowledge-db/performance/2026_recommendations.md (자산군 방향 제안 append)
forbidden:
  - knowledge-base/portfolio/user_portfolio.md 읽지 않음 (개인 데이터 격리)
  - 구체적 종목·ETF 티커 지정 금지 (자산군 수준만)
template: rebalancing
```

briefing-lead 워크플로 + KB portfolio/rebalancing_history.md append 강제.
