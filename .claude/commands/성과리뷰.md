$ARGUMENTS 인자로 **성과 리뷰 (C-9 단독)** 를 실행해줘.

## 명령 정보

- **모듈:** C-9 단독 분리 실행 (briefing_module_CDE.md)
- **산출물:** `reports/briefing/performance_review_{기간}_{YYYYMMDD}.html`
- **차이 vs /주간리포트:** 주간리포트는 C-1~C-10 + C-9 부속, 성과리뷰는 C-9 단독 + 더 긴 기간

## 인자 해석

- 인자 없음 → `1m` (최근 1개월 ≒ 22영업일)
- `1w` → 최근 1주
- `2w` → 최근 2주
- `1m` → 최근 1개월
- `3m` → 최근 3개월
- `--html` → HTML 생성 (기본)

## briefing-lead 에 전달할 컨텍스트

```
mode: performance_review
period: 1w | 2w | 1m | 3m
sections:
  ## 종합 통계
  - 전체 제안 N건 / 적중 N건 / 오류 N건 / 진행중 N건
  - 종합 적중률 __%
  - 직전 기간 대비 변화 (±__%p)

  ## 적중률 분해 — 모듈별
  | 모듈 | 제안 | 적중 | 오류 | 진행중 | 적중률 |
  | morning / evening / weekly / crypto / portfolio / global / rebalancing |

  ## 적중률 분해 — 카테고리별
  | 카테고리 | 제안 | 적중률 | (종목 / ETF / 토큰 / 자산군 / 시나리오 / 이벤트)

  ## 적중률 분해 — 시간축별
  | 시간축 | 제안 | 적중률 | (단기 1~3개월 / 중기 3~12개월 / 장기 1~5년)

  ## 가장 정확했던 판단 (Top 3)
  ## 가장 빗나간 판단 (Top 3)
  ## 교훈 노트 (3~5개)
required_actions:
  - knowledge-db/performance/2026_recommendations.md 읽기 (대상 기간)
  - knowledge-db/market/2026_daily_prices.md 읽기 (실측 가격)
  - 13F 제안은 다음 분기 공시 데이터 대조
  - 적중 판정: 방향 일치 + 변동률 > 1%
  - 오류 판정: 부호 반대 + 변동률 > 1%
  - 진행중: 변동률 ≤ 1% 또는 시간축 미도래
  - knowledge-db/performance/2026_hit_rate.md append (모듈/카테고리/시간축 분해 통계 1행씩)
forbidden:
  - 데이터 없는 적중률 추정 (knowledge-db 양쪽 실측 필수)
  - 자기 정당화 (#14)
  - 시나리오 미확정 항목을 적중·오류로 분류
extras:
  - debate-card 생략 가능
  - contrarian-card 생략 가능
  - 4종 포트폴리오 방향 생략 가능
template: performance_review
```

briefing-lead 워크플로. knowledge-db/performance/2026_hit_rate.md append 강제.
