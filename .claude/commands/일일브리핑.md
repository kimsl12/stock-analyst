argument-hint: [YYYYMMDD] [--skip-collect] [--skip-kb]
description: 일일 브리핑(시장·매크로·거물 통합)을 생성합니다. 예) /일일브리핑 / /일일브리핑 20260407 --skip-collect
agent: briefing-synthesizer
---

$ARGUMENTS 인자로 일일 브리핑 파이프라인을 실행해줘.

## 인자 해석 규칙

- 인자 없음 → 대상일 = **오늘 KST** (`date '+%Y%m%d'` 결과)
- `YYYYMMDD` 형태 토큰 1개 → 그 날짜를 대상일로 사용
- `--skip-collect` 플래그 → Phase 0-A의 **market-data-collector 호출 생략**
  (이미 오늘자 knowledge-base/market/ 가 최신인 경우만)
- `--skip-kb` 플래그 → Phase 0-A의 **kb-updater 호출 생략**
  (이미 오늘자 knowledge-base/macro/ 가 최신인 경우만)
- 두 플래그 동시 사용 가능

## 실행 호출 순서 (절대 위반 금지)

```
[Phase 0-A] 데이터 수집 (직렬 — 선행)
   ① market-data-collector   (--skip-collect 시 생략)
        ↓ knowledge-base/market/ + knowledge-db/market/
   ② kb-updater              (--skip-kb 시 생략)
        ↓ knowledge-base/macro/ , knowledge-base/industry/

[Phase 0-B] 분석 (병렬 — 3개 동시 호출, Task 도구)
   ③a market-analyst → analysis/briefing/market_analysis_{YYYYMMDD}.md
   ③b macro-analyst  → analysis/briefing/macro_analysis_{YYYYMMDD}.md
   ③c guru-analyst   → analysis/briefing/guru_analysis_{YYYYMMDD}.md

[Phase 0-C] 통합 (직렬 — 본 에이전트가 직접 수행)
   ④ briefing-synthesizer  → reports/briefing/daily_briefing_{YYYYMMDD}.md
```

## 단계별 지침

### Step 1. 사전 점검
- `mkdir -p analysis/briefing reports/briefing` 실행
- 대상일(`{YYYYMMDD}`) 결정 후 출력
- `--skip-collect` / `--skip-kb` 여부 출력

### Step 2. Phase 0-A — 데이터 수집 선행 (스킵 플래그 없으면)
- Task 도구로 `market-data-collector` 호출
  ```
  오늘 ({YYYY-MM-DD}) 시장 스냅샷을 수집해줘.
  knowledge-base/market/ 5개 파일을 갱신하고
  knowledge-db/market/{YYYYMMDD}.jsonl 에 축적해.
  ```
- 완료 후 Task 도구로 `kb-updater` 호출
  ```
  오늘({YYYY-MM-DD}) 일일 브리핑용 매크로 KB를 갱신해줘.
  sector: (없음 — 매크로 전반)
  macro_tags: ["통화정책", "지정학", "공급망", "에너지"]
  knowledge-base/macro/ 의 valid_until 이 오늘 이후가 아닌 파일을 우선 갱신해.
  ```
- 두 호출 모두 완료 후 `ls -la knowledge-base/market/ knowledge-base/macro/` 로 확인

### Step 3. Phase 0-B — 3개 분석가 병렬 호출
**한 번의 응답에서 Task 도구를 3회 동시 호출해야 한다 (병렬).**

- `market-analyst` 호출
  ```
  대상일: {YYYYMMDD}
  knowledge-base/market/ 의 오늘자 스냅샷을 읽고
  analysis/briefing/market_analysis_{YYYYMMDD}.md 를 작성해.
  매수·매도 추천 절대 금지. 관찰·해석만.
  ```
- `macro-analyst` 호출
  ```
  대상일: {YYYYMMDD}
  knowledge-base/macro/ 를 읽고
  analysis/briefing/macro_analysis_{YYYYMMDD}.md 를 작성해.
  통화정책·지정학·공급망 핵심 + 매크로 위험 등급(1~5) 포함.
  ```
- `guru-analyst` 호출
  ```
  대상일: {YYYYMMDD}
  knowledge-base/market/guru_positions.md 와 reference/guru_watchlist.md 를 읽고
  analysis/briefing/guru_analysis_{YYYYMMDD}.md 를 작성해.
  ⚠️ 13F 시차(분기말 기준 최대 45일) 경고를 본문 상단에 명시할 것.
  ```

### Step 4. Phase 0-B 산출물 검증
- `ls -la analysis/briefing/` 로 3개 파일 모두 존재 확인
- 누락 시 해당 분석가만 1회 재호출 (같은 프롬프트)
- 2회 연속 누락 시 사용자에게 보고 후 중단

### Step 5. Phase 0-C — briefing-synthesizer (본 에이전트가 직접 수행)
- 본 명령어의 `agent:` 가 이미 `briefing-synthesizer` 이므로
  Step 5는 본 에이전트의 워크플로 그대로 진행
- `.claude/agents/briefing-synthesizer.md` 의 워크플로 9개 단계를 순서대로 실행
- 자가 검증 (Top 3 정확히 3개, 13F 시차 고지, 새 수치 0건, 한국어)
- 최종 파일 저장: `reports/briefing/daily_briefing_{YYYYMMDD}.md`

### Step 6. 사용자 보고
- 생성된 파일 경로 출력
- Top 3 액션 아이템 한 줄 요약
- 데이터 시차 고지 (시장 / 매크로 / 13F)
- 생성 완료 후 `git add reports/briefing/ analysis/briefing/` → commit → `git pull --rebase origin main` → `git push origin main` 자동 수행

## 절대 금지 사항

1. ❌ 매수·매도·익절·손절·비중조정·목표가·손절가 표현
2. ❌ knowledge-base/portfolio/ 직접 읽기
3. ❌ analysis/briefing/ 의 다른 분석가 산출물을 분석가가 서로 읽기
4. ❌ Top 3 액션 아이템이 아닌 다른 개수
5. ❌ 13F 시차 고지 누락
6. ❌ 영어 본문 작성
7. ❌ main 외 브랜치로 push (reports/briefing 산출물은 main 직접 push)
