agent: briefing-lead
---

$ARGUMENTS 인자로 **내 포트폴리오** 를 실행해줘.

## 명령 정보

- **목적:** 사용자 1인의 개인 보유 자산 관리 (공개 콘텐츠가 아님)
- **산출물:** `reports/briefing/user_portfolio_{YYYYMMDD}.html`
- **격리:** 다른 모든 명령은 `knowledge-base/portfolio/user_portfolio.md` 를 읽지 않는다.
  본 명령만 유일한 진입점.

## 인자 해석

- 인자 없음 → **인터랙티브 모드** (미등록 항목 입력 요청)
- `--view` → 조회만 (입력 받지 않음)
- `--html` → HTML 생성 (기본)

## briefing-lead 에 전달할 컨텍스트

```
mode: user_portfolio
interactive: true | false (--view 시 false)
sections:
  ### 투자자 프로파일 (사용자 입력)
  - 투자 성향 (안전 / 중립 / 공격 / 배당)
  - 총 투자 가능 금액
  - 투자 기간 (1년 미만 / 1~3년 / 3~5년 / 5년+)
  - 관심 테마

  ### 보유 종목 (주식 + ETF)
  | 티커 | 종목명 | 유형 | 시장 | 보유 수량 | 평균 매수가 | 현재 비중 | 비고 |

  ### 보유 자산군 요약
  | 자산군 | 현재 비중 | 목표 비중 | 괴리 |

  ### 4종 모델 포트폴리오와 비교
  - 사용자 보유 vs 4종 중 가장 가까운 유형
  - 자산군별 차이 (현재 vs 목표 모델)
  - 매크로 환경 기반 권장 조정 방향 (자산군 수준만)
required_actions:
  - knowledge-base/portfolio/user_portfolio.md 읽기
  - interactive=true 일 때 미등록 항목 input 요청
  - 사용자 입력 → user_portfolio.md 갱신 (briefing-lead 만 쓰기 가능)
  - 4종 모델 비교 시 model_portfolios.md 읽기
forbidden:
  - HTML 출력 시 개인 데이터 평문 노출 (이름·계좌번호 등)
    → 보유 종목명·티커·비중까지만 표시
  - 다른 사용자에게 본 산출물 공유 가정 금지
  - debate-card / contrarian-card 생략 가능
  - 13F 시차 경고 불필요
template: user_portfolio
git_handling:
  - user_portfolio.md 는 git 추적 (현재 .gitignore 미설정)
  - 단, 향후 .gitignore 에 추가 검토 필요 (개인 데이터)
```

briefing-lead 워크플로. 인터랙티브 입력 → user_portfolio.md 갱신 → 4종 모델 비교.
