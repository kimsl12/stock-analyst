$ARGUMENTS 인자로 **풀 브리핑 (A+B+C+E)** 를 실행해줘.

## 명령 정보

- **포함 모듈:** A + B + C + E (모닝 + 이브닝 + 주간 + 크립토)
- **포함 안 함:** F (모델포트폴리오), G (글로벌인텔리전스) — 별도 명령
- **산출물 4개:**
  - `reports/briefing/morning_{YYYYMMDD}.html`
  - `reports/briefing/evening_{YYYYMMDD}.html`
  - `reports/briefing/weekly_{YYYYMMDD}.html`
  - `reports/briefing/crypto_{YYYYMMDD}.html`

> ⚠️ **명칭이 "풀"이지만 모든 모듈을 의미하지 않음.** F·G 는 별도 명령으로 실행 (briefing_rules_commands.md 명세).

## 인자 해석

- 인자 없음 → 오늘 KST
- `YYYYMMDD` → 해당 일자
- `--skip-collect` → 1회만 (4편 공유)
- `--html` → 4편 모두 HTML 생성 (기본)

## briefing-lead 에 전달할 컨텍스트

```
mode: full
target_date: {YYYYMMDD}
core_efficiency:
  - Phase 0-A (데이터 수집) 1회만 — 4편 공유
  - Phase 0-B (분석가 호출) 1회만 — global-macro-analyst, correlation-monitor 결과 4편에서 재사용
  - Phase 0-C (briefing-lead 종합) 4회 — morning → evening → weekly → crypto 순차
sections:
  - 각 모듈은 해당 명령 (/모닝, /이브닝, /주간, /크립토) 의 sections 동일
required_extras:
  - 각 편마다 debate-card / contrarian-card 1건씩 (총 4 + 4 = 8 카드)
  - 4종 포트폴리오 방향 (morning/evening/weekly 3편)
  - 13F 시차 경고 (morning/evening/weekly 3편)
fallback:
  - 토큰 한도 우려 시 자동 폴백: weekly → crypto → evening → morning 순서로 단순화
required_actions:
  - global-macro-analyst (mode=quick + weekly 결합)
  - correlation-monitor (mode=full)
  - briefing-report-generator 4회 호출 (template=morning, evening, weekly, crypto)
template: full
git_commit:
  - 4개 산출물을 단일 커밋으로: "feat(briefing): full briefing {YYYY-MM-DD} (A+B+C+E)"
```

briefing-lead 워크플로 4번 (효율화 + 토큰 폴백).
